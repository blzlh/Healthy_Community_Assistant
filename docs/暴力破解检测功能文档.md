# 暴力破解检测功能文档

## 功能概述

本功能实现了完整的暴力破解检测与防护机制，符合毕业设计要求：

- **检测规则**：同一 IP 在 60 秒内登录失败 ≥ 5 次
- **自动处置**：触发告警并自动封禁该 IP 10 分钟
- **日志记录**：完整的登录尝试记录和安全事件日志
- **可视化监控**：管理员可查看安全事件和封禁列表

## 功能测试方法

### 测试一：暴力破解检测

#### 方法 1：使用脚本模拟（推荐）

创建测试脚本 `test-brute-force.sh`：

```bash
#!/bin/bash

# 测试暴力破解检测
# 需要先安装 jq: apt-get install jq 或 brew install jq

API_URL="http://localhost:3001"
TEST_EMAIL="test@example.com"

echo "=== 开始暴力破解测试 ==="
echo "目标 API: $API_URL"
echo "测试邮箱: $TEST_EMAIL"
echo ""

# 连续发送 6 次失败登录请求
for i in {1..6}; do
  echo "第 $i 次尝试..."
  
  response=$(curl -s -X POST "$API_URL/auth/login" \
    -H "Content-Type: application/json" \
    -d "{\"email\": \"$TEST_EMAIL\", \"code\": \"wrong-code\"}" \
    -w "\nHTTP_STATUS:%{http_code}")
  
  http_code=$(echo "$response" | grep "HTTP_STATUS:" | cut -d':' -f2)
  body=$(echo "$response" | grep -v "HTTP_STATUS:")
  
  echo "  HTTP 状态: $http_code"
  
  if [ $i -ge 5 ]; then
    echo "  应该触发 IP 封禁"
    # 检查是否被封禁
    blocked_response=$(curl -s "$API_URL/auth/login" \
      -H "Content-Type: application/json" \
      -d "{\"email\": \"$TEST_EMAIL\", \"code\": \"test\"}")
    
    if echo "$blocked_response" | grep -q "封禁"; then
      echo "  ✅ IP 已被封禁！"
      break
    fi
  fi
  
  sleep 1
done

echo ""
echo "=== 测试完成 ==="
```

运行测试：

```bash

./test-brute-force.sh
```

```typescript
{
  todayFailedAttempts: number;      // 今日登录失败次数
  activeBlockedIps: number;         // 当前封禁 IP 数量
```

## API 接口文档

### 获取安全统计

```http
GET /security/statistics
Authorization: Bearer <token>
```

### 获取登录尝试记录

```http
GET /security/login-attempts?ipAddress=<ip>&limit=100
Authorization: Bearer <token>
```

### 获取封禁 IP 列表

```http
GET /security/blocked-ips?limit=100
Authorization: Bearer <token>
```

### 封禁 IP

```http
POST /security/block-ip
Authorization: Bearer <token>
Content-Type: application/json

{
  "ipAddress": "192.168.1.1",
  "reason": "手动封禁",
  "durationMinutes": 60
}
```

### 解封 IP

```http
DELETE /security/block-ip/<ipAddress>
Authorization: Bearer <token>
```

### 获取安全事件日志

```http
GET /security/logs?type=brute_force&severity=high&resolved=false
Authorization: Bearer <token>
```

### 解决安全事件

```http
POST /security/logs/<logId>/resolve
Authorization: Bearer <token>
```

## 注意事项

1. **生产环境建议**：
   - 使用 HTTPS
   - 配置 Nginx 限流作为第一道防线
   - 设置日志轮转避免日志过大
   - 监控 Redis 内存使用

2. **性能考虑**：
   - Redis 作为第一层缓存，响应快
   - 数据库作为持久化存储，可靠性高
   - 已创建索引优化查询性能

3. **合规要求**：
   - 所有测试应在本地或隔离环境进行
   - 日志中不应包含用户密码等敏感信息
   - 遵守数据保留政策
